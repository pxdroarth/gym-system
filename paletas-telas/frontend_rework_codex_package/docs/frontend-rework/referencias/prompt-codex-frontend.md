IMPORTANTE:
- Não reescreva o sistema do zero.
- Não quebre contratos existentes.
- Preserve auth, escopo tenant/unit, permissões visuais, financeiro separado e consolidado read-only.
- Use as referências visuais em `/docs/frontend-rework/telas` e `/docs/frontend-rework/paleta` como direção obrigatória.
- Trate as imagens como alvo visual do produto real, não como liberdade estética total.

1. diagnosticar arquivos afetados
2. listar componentes reaproveitáveis

4. explicar lacunas entre mockups e sistema atual
5. só então implementar
